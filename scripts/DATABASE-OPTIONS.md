## Why This Approach is Better

**30,000 plants sounds impressive, but:**
- 90% of users only search for ~200 common plants
- Quality data for 1,000 plants > poor data for 30,000
- Smaller file = faster app load
- Easier to maintain and update

**Better strategy:**
1. Start with curated 500-1,000 plants with excellent data
2. Add "Search Online" button for rare plants (uses API only when needed)
3. Learn which plants users actually search for
4. Expand database based on real usage
